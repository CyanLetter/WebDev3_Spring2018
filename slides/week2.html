What is Sass

Syntactically Awesome Stylesheets

Sass is a preprocessor

Preprocessing

Taking one file and spitting out another

In our case, taking .scss files and turning them into .css

But how?

Command line tools, like gulp or grunt

Apps like Prepros or Codekit

Why Sass?

Allows you to better organize your styles in large projects.

Gives new, exciting features to vanilla css.

Variables

Assign a value to a particular word.

Change styles in one place.

$

The dollar sign means 'this is a variable'


$primary-color: #aa0045;

.main-container {
  background: $primary-color;
}

Math!


$column-1-3: 1 / 3 * 100%;
$column-1-1: 100%;

.text-col {
	width: $column-1-3;
}

Sass can handle basic math and some special functions.

More examples



p {
  width: $column-1-1;
}

// Switch to a three column layout
@media only screen and (min-width: 768px) { 
  p {
    width: $column-1-3;
  }
}

Guidelines

  • Make variables reusable
  • Don't make one until you know it will help
  • Keep variables together

Nesting

Putting selectors inside of selectors.

Helps organize code into modules and sections.


/* this */
.parent {

  .child {
    background: red;
  }
}

/* is the same as this */
.parent .child {
  background: red;
}

Complex Nesting

& means 'and also'

> means 'direct child'


/* Applies to parent that also has red class */
.parent {
	
  &.red {
    background: red;
  }
}
						

/* Apply to all direct children */
.someclass {
  >* {
    max-width: 640px;
    margin: 10px auto;
  }	
}

Guidelines

  • Don't over specify
  • Apply general rules first
  • Check your output

@-Rules

Also known as Directives.

So named because they are prefixed with an @ symbol

How they work

@-Rules tell Sass to do something special when processing.

Very similar to @ directives in vanilla css.

Basic Directives

  • @extend
  • @mixin
  • @media
  • @include

@media

Acts like standard media queries.

Can be nested inline

Basic Example


.container {
  width: 60%;
  @media (min-width: 200px) and (max-width:600px) {
    width: 100%;
  }
}

Combine with variables to make responsive modules.

Note that these are mobile-first breakpoints.


$tablet-breakpoint: 640px;
$desktop-breakpoint: 960px;

.content {
  width: 100%;

  @media (min-width: $tablet-breakpoint) {
    width: 50%;
  }
  @media (min-width: $desktop-breakpoint) {
    width: 30%;
  }
}

Guidelines

Some people like this style, some hate it.

  • Pro: define all styles for a selector in one place.
  • Con: Media queries are no longer in one central place.
  • Try to use with variables and other centralized components.
  • Use sparingly.

The Rest

extend, import, and mixins all get more helpful as projects increase in size.

@extend

Repeat styles you've already written.

Duplicates class names.

Some people don't like them

Make extendable modules


.module {
	height: 120px;
	background: #333;
}

.module-variant {
	@extend .widget;

	background-image: url('myImage.jpg');
}

Make placeholders with %


%button {
	height: 60px;
	width: 100px;
	color: white;
}

.submit {
	@extend %button;

	background: green;
}

Guidelines

  • Centralize generic behaviors.
  • Base modules are good places to define media queries, for instance.

@import

Merge partial files into your main one.

Can act like a css import statement

Defining a partial

Sass sees a leading underscore as a partial file, e.g.

  • main.scss is a regular file
  • _part.scss is a partial

However, you do not have to include the underscore. It is simply an organizational tool.

If you have two files, main.scss and _part.scss, you can merge them.


/* inside of main.scss */
.main-styles {
	/* Style rules here */
}

/* Partial will be added here */
@import "part";

Guidelines

  • Be careful where you import, styles still cascade!
  • Use this for organization.
  • You could separate variables or modules.
  • You can separate different parts of your site.

@mixin, @include

Code you define once, reuse anywhere.

Duplicates style rules.

Can execute complex instructions.

Usage

Define code with @mixin

use code with @include

Basic use


@mixin callout-text {
	font-size: 20px;
	font-weight: bold;
	text-align: center;
}

.widget-callout {
	@include callout-text;
}

Breakpoints

Even simpler inline breakpoints


$tablet-breakpoint: 640px;
$desktop-breakpoint: 960px;

@mixin bp-large {
  @media only screen and (min-width: $desktop-breakpoint) {
    @content; // This lets you put stuff inside of the mixin
  }
}

@mixin bp-small {
  @media only screen and (min-width: $tablet-breakpoint) {
    @content;
  }
}

Usage


.main-column {
  width: 100%;

  @include bp-small {
    width: 80%;
  }

  @include bp-large {
    width: 50%;
  }
}

Mixins are everywhere

A quick search for 'Sass mixins' will yield many recipes that people have already written.

Some even ship Sass mixin libraries, like Bourbon or Sassmatic

Adding gulp plugins

Let's add something to automatically prefix files.

We will be editing gulpfile.js

Find it on npmjs.com

Download the plugin

Run the following command:


npm install --save-dev gulp-autoprefixer

This downloads and adds to package.json

Add to gulpfile

A little different since our gulpfile is already set up. Add the following to the sass task:


.pipe(autoprefixer({
    browsers: ['last 2 versions'],
    cascade: false
}))

Try it out!

Restart gulp and add a transform to your sass file.
