Accesibility

Accessibility is...

Making sure that anyone can use your site, under any condition.

Why

  • Good accessibility is good user experience
  • Not difficult, but frequently an afterthought
  • Government requirements (sometimes)

Who

Anyone who has an impairment, be it temporary or permanent:

  • Vision
  • Hearing
  • Motor Skills
  • Cognitive Ability
  • Hardware

Standards

  • WCAG, the official W3 guielines, and the WebAIM checklist
  • The A11Y project
  • Government guidelines like Section 508

The bare minimum

Focus

When navigating with a keyboard, focus is the currently selected element.

  • Only certain elements are focusable
  • DOM order determines focus order

Offscreen Elements

Even if offscreen, elements can gain focus.

Use
display: none;
or
visibility: hidden;
to prevent this.

Can use libraries like inert.js

Tab Index

Use to set focus order when you can't move the html.

  • Like z-index, use sparingly
  • DOM order determines focus order
  • Tabindex > 0 sets priority
  • Tabindex of -1 removes element from focus order
  • You may need to manage focus with javascript

Semantics

Giving inherent meaning to your page with HTML elements. See this demo page.

Breakdown

Elements contain different information

  • Role / Type
  • Name
  • Value
  • State (Checked, unchecked, etc)

Eamples include headings, figures, and inputs

Alternate Text

  • Screen readers will use file names by default
  • Describe images with the alt attribute
  • Skip images, like icons, that don't need description with alt=""

Styles

Contrast, Size, and Responsiveness

Responsiveness

Make your site work on all devices.

  • Mobile support is part of accessibility
  • Load time is part of accessibility
  • Use a proper viewport meta tag

Size

Make stuff big, let people make it bigger.

  • Use em and rem instead of px
  • #1 complaint about the web is bad typography
  • Give buttons reasonable size (48px) and spacing (32px)

Contrast

Make text easy to read. Use a contrast checker.

  • Applies to elderly and colorblind
  • Use pattern and shape for important elements
  • Always use a focus style
